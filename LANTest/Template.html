LAN Test Utility

Ask your friends to visit {{Address}} in their browsers (use Windows command `ipconfig` for more available IP address options).

If they cannot see this page, consider one of the following (in order of relevance - latter items are less relevant):

  1. You might need to TURN OFF (yes, turn off completely instead of mess with inbound/outbound rules) Windows Firewalls. This is inside Control Panel\System and Security\Windows Defender Firewall\Customize Settings.
    • Notice surprising enough, even if you are pretty darn sure you are on a Private Network, you might need to turn off Public Network firewall for it to work. Hopefully that's all there is to hurdle your connection within your LAN.
    • Notice only the host needs to turn off firewall, and all peers usually don't need to change any network configuration on their side.
    • For specific ways to configure Firewall instead of turning it off completely, see relevant discussion here: Hamachi Inbound Connection
  2. The interconnection between arbitary machines might not be permitted within your current network;
  3. Your computer might not be discoverable on the network (e.g. you are running Windows in a public instead of private network):
    • Check Control Panel\All Control Panel Items\Network and Sharing Center\Advanced sharing settings for Network Sharing options.

Otherwise, there are configuration issues with your network router and it's beyond what this program can do. Try things like "Port Forwarding" among other solutions. (Most likely the router just doesn't allow LAN connection, consider using a different router)
In particular, if none of the games you have can be played in LAN mode, then you need to figure out how to replace/setup your router - and it might be related to ISP specific routers. Good luck!


P.S. Worth noting though, as mentioned in Point 1 above - for some reason I don't know yet, even if ISP's router doesn't seem to work with Port Forwarding and switching a router (e.g. change from Bell HomeHub 3000 to TP-Link Archer, without changing any other configuration on the program) seems to solve the problem - in the former case by turning off Windows Firewall it can also solve the problem (i.e. even with original Bell HomeHub 3000 router!). Anyone more informed in this subject might be interested in adding why to this page.


P.S. In other cases, consider using a VPN network application like LogMeIn Hamachi to establish LAN connections between different devices. Or if you have a better open source alternative, please don't hesitate to email me and I will update the recommendation on this page.
